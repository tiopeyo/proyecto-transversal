Proyecto Transversal

Desarrollo de Soluciones de Software Completas

🎯Objetivo Principal

Desarrollar una solución de software completa y funcional para un cliente simulado, operando en un servidor local configurado por los estudiantes, aplicando conocimientos de Diseño y Desarrollo de Software, Análisis y Administración de Sistemas Operativos, y Análisis de Redes y Protocolos de Comunicación.

⏱️Estructura Temporal

FASE 1
Análisis
FASE 2
Backend
FASE 3
Frontend
1

Análisis y Diseño

  • Documento de Definición profesional
  • Levantamiento de Requisitos
  • Diseño de Base de Datos (MER y MR)
  • Mockups Frontend (UI/UX)
  • Plan de Infraestructura
2

Infraestructura y Backend

  • Montaje del Servidor Virtualizado
  • Implementación de Base de Datos
  • Desarrollo del Backend (API)
  • Configuración de Red y Firewall
3

Frontend y Despliegue

  • Desarrollo del Frontend
  • Sistema de Seguridad (Login, Roles)
  • Implementación HTTPS
  • Pruebas y Puesta en Marcha

📋Pautas Generales

🗓️ Planificación de Tiempos

Crear y mantener una Carta Gantt simple para planificar tareas, asignar responsables y establecer plazos internos para las tres fases del proyecto.

📝 Documentación del Desarrollo

Generar un documento técnico formal desde la Fase 1 que evolucione con el proyecto, registrando desde requisitos iniciales hasta el diseño final.

💻 Entorno Técnico

Configurar un servidor local con sistema operativo, servicios web, base de datos e implementar protocolos de seguridad, incluido HTTPS.

🎯 Entregables

Cada fase tiene entregables específicos que deben cumplir con estándares profesionales de calidad y funcionalidad completa.
